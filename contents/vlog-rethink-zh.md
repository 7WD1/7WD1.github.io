可解释性这个词，在大模型时代需要重新定义。不是因为以前的定义错了，是因为研究对象变了。

以前我们解释一个 ResNet，任务是明确的：这张图为什么被分类成猫？特征归因给出答案——像素级的贡献度。解释和预测之间有清晰的映射关系。但 LLM 改变了游戏规则。一个语言模型的输出不是单一的分类标签，是一段可能包含推理、创意、事实检索的文本。你解释什么？解释整个生成过程？还是某个特定 token 的选择？

这篇论文的答案不算惊艳，但它问对了问题。

第一个是"解释粒度"的问题。传统 XAI 的粒度是特征级别——哪个输入维度最重要。但 LLM 的输入是 token 序列，输出也是 token 序列。特征归因在 token 级别几乎没用——"the" 这个 token 对输出的贡献度高，这能告诉你什么？需要更粗的粒度：概念级、语句级、甚至是推理步骤级。

第二个是"忠实度"的定义本身。在小模型时代，忠实度可以通过输入扰动来验证——改变输入的某个特征，看输出是否按预期变化。但 LLM 的输入空间太大，扰动空间指数级增长。穷举验证不可能，采样验证不可靠。忠实度变成了一个无法严格定义的概念。

第三个是"可解释性的受众"问题。工程师需要的技术解释和普通用户需要的解释完全不同。以前的 XAI 方法默认受众是工程师。LLM 时代，解释的受众变成了所有人——包括不懂技术的终端用户。这意味着解释必须用自然语言，但自然语言解释的忠实度天生就低。

可解释性不是一个单一的技术问题，而是一组问题的集合。不同的模型规模、不同的应用场景、不同的受众，需要不同形式的解释。一刀切的"可解释性"框架在大模型时代行不通了。

我的补充观点是：我们可能需要放弃"完美解释"这个目标。小模型时代我们追求"每个预测都有忠实解释"。LLM 时代，这个目标既不可行也不必要。用户不需要知道模型为什么选了"however"而不是"but"。他们需要知道的是：模型的回答是否可靠？在什么条件下可能出错？如何验证？

从"解释每个预测"到"建立整体信任"——这才该是可解释性研究的新方向。这篇论文是少数几个意识到这个转变的作品。
